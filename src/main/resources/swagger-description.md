REST API для управления банковскими картами с JWT-аутентификацией и ролевым доступом.

## Аутентификация
* Регистрация через `POST /api/auth/register` — возвращает JWT токен
* Вход через `POST /api/auth/login` — возвращает JWT токен
* Все защищённые endpoints требуют заголовок `Authorization: Bearer <token>`

## Уровни доступа
* `ROLE_USER` — просмотр своих карт, переводы между своими картами, запрос на блокировку
* `ROLE_ADMIN` — создание/удаление карт, управление пользователями, обработка заявок на блокировку

## Формат ответов
Все endpoints возвращают `CommonResponse<T>`:
```json
{
  "data": {},
  "status": "SUCCESS",
  "timestamp": "2026-04-03T22:18:39"
}
```

Ответ с ошибкой:
```json
{
  "status": "ERROR",
  "problemDetail": {
    "type": "about:blank",
    "title": "Not Found",
    "status": 404,
    "detail": "Card not found"
  },
  "timestamp": "2026-04-03T22:18:39"
}
```

## Коды ошибок
* `400` — ошибка валидации или неверный формат запроса
* `401` — токен отсутствует или истёк
* `403` — недостаточно прав или аккаунт заблокирован
* `404` — карта или пользователь не найден
* `409` — конфликт (пользователь уже существует, заявка на блокировку уже создана)
* `422` — недостаточно средств для перевода
* `500` — внутренняя ошибка сервера
